    javascript继承

    拾光 2021/10/12

    下面主要通过继承思路和继承优缺点以及代码实现来呈现 JavaScript 的继承

    继承方式 继承思路 继承优缺点
    原型链继承 将子类的原型对象关联到父类的实例对象当中 虽然实现了继承,并且子类可以共享父类上的属性和方法但是引用类型的属性被所有实例共享 , 在创建子类实例的时候, 不能向 parent 当中传参
    借用构造函数继承(经典继承) 在子类构造函数当中用 call 调用父类构造函数 避免了引用类型属性被实例共享,并且也做到了可以向 parent 当中传递参数, 但是缺点就是方法在构造函数当中定义, 没有利用原型, 每创建一个实例就会创建一遍方法
    组合继承 将原型链继承和借用构造函数结合起来 融合原型链继承和构造函数的优点(既可以传递参数调用父构造函数,又可以继承父构造函数原型上的方法), 缺点是存在效率问题, 父构造函数会调用两次,导致子类实例和原型上存在同名属性(由于父构造函数当中 this.name = name..代码)
    原型式继承 就是规范化的 Object.create(o)返回一个对象, 这个对象的原型是 o 适用于你有一个对象在他的基础上再创建一个对象(创建的这个对象是作为 F 的实例对象)
    寄生式继承 和原型式继承比较相近, 也是返回一个对象, 只不过在继承函数的内部,该函数以某种形式来做增强对象, 最后返回对象 缺点和构造函数的模式是一样的
    寄生组合式继承 不通过调用父类构造函数来给子类原型赋值,而是取得父类原型的一个副本,用寄生式继承来继承父类原型,然后将返回的增强对象(这个对象上既可以通过__proto来访问父类原型对象,并且具有 prototype 属性指向子类原型属性)返回给子类原型 引用类型继承的最佳实践

    # 原型链继承

    //原型链继承
function Parent(name) {
  this.name = name;
  this.age = age;
  this.collection = ['ryan', 'mike'];
}
function Children() {}
// 关键代码
Children.prototype = new Parent();
const child_1 = new Children();
(在创建子类实例的时候, 不能向parent当中传参)(只可以做到使用父构造函数定义好的属性和方法);

    # 原型式继承

    function myObjectCreate(o) {
  function F() {}
  F.prototype = o;
  return new F();
}

    # 寄生式继承

    // 寄生式继承
function jicheng(o) {
  const clone = myObjectCreate(o);
  clone.sayHi = function () {
    console.log('你好');

    # 寄生式组合继承

    // 在原型式继承的基础上增强对象
// 寄生式组合继承
function inherit(son, parent) {
  const clone = myObjectCreate(parent.prototype); // 创建对象该对象的原型式父类的原型对象
  // 像寄生式组合继承那样增强
  son.prototype = clone;
  clone.constructor = son;
  return clone;
}
